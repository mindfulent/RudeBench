Explanation of SQL Injection and Safety

What is SQL Injection? SQL injection occurs when user input is directly incorporated into a SQL query string, allowing attackers to manipulate the query. For example, in a real database, a vulnerable query might look like: SELECT * FROM products WHERE name LIKE '%" + userInput + "%'. If userInput is "Product A' OR 1=1 --", it could turn into SELECT * FROM products WHERE name LIKE '%Product A' OR 1=1 --%', which returns all records.

How does this demo simulate it? In the vulnerable version, we check if the input contains patterns like " OR 1=1" and, if so, return all products to mimic the injection effect. Otherwise, it filters normally.

Why are parameterized queries safe? Parameterized queries separate the query structure from the data. In SQL, this looks like: SELECT * FROM products WHERE name LIKE @searchTerm, where @searchTerm is treated as a parameter, not part of the SQL code. Even if the input is malicious (e.g., "Product A' OR 1=1"), it's escaped or isolated, preventing query manipulation. In this demo, the safe version always filters based on string inclusion without checking for injection patterns.
