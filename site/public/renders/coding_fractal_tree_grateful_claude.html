🔍 How the Recursive Algorithm Works

Each call to drawBranch(x, y, length, angle, depth) does three things:

  1. Draw a line segment from (x, y) in the given direction.
  2. Recurse left – rotate by −25 ° and reduce length by 30 %.
  3. Recurse right – rotate by +25 ° and reduce length by 30 %.

The base case is depth === 0, at which point the function returns immediately without drawing. This guarantees the recursion always terminates. 

function drawBranch(x, y, len, angle, depth) {
  if (depth === 0) return;          // base case – stop recursing

  const x2 = x + Math.cos(angle) * len;
  const y2 = y + Math.sin(angle) * len;

  ctx.beginPath();
  ctx.moveTo(x,  y);
  ctx.lineTo(x2, y2);
  ctx.stroke();

  drawBranch(x2, y2, len * 0.7, angle - 0.436, depth - 1); // left
  drawBranch(x2, y2, len * 0.7, angle + 0.436, depth - 1); // right
}

📐 Understanding Recursion Depth Limits

At depth d a binary-branching tree produces 2d+1 − 1 total branches. The table below shows how quickly this grows: 

Depth │ Branches │ Max simultaneous stack frames
──────┼──────────┼──────────────────────────────
  1   │        1 │   2
  4   │       31 │   5
  7   │      255 │   8
 10   │    2 047 │  11
 12   │    8 191 │  13
 20   │1 048 575 │  21   ← never do this recursively!

Notice that the branch count grows exponentially, but the maximum stack depth only grows linearly — it equals depth + 1. This is because the two recursive calls are sequential, not simultaneous: the left subtree is fully explored before the right one begins, so only one path from root to leaf is on the call stack at any moment.

💥 Stack Overflow Risks

Every function call in JavaScript (and most languages) pushes a stack frame onto the call stack. A stack frame stores:

  • The function's local variables (x, y, len, …)
  • The return address — where execution resumes after the call
  • Internal bookkeeping used by the JavaScript engine

Browser engines typically allow between 10 000 and 15 000 nested frames before throwing a RangeError: Maximum call stack size exceeded.

For our tree the maximum simultaneous depth is only depth + 1 (≤ 13 here), so there is no real risk in this demo. However, if you accidentally removed the base case, or if the branching factor were larger (e.g. a ternary tree at depth 20), you would hit the limit quickly.

Common causes of accidental stack overflow:

  • Missing or unreachable base case
  • Recursive calls on data structures with cycles (graphs without a visited set)
  • Mutual recursion (A calls B calls A …) with no termination
  • Very deep trees / linked lists processed node-by-node

🛠️ Practical Solutions

1 — Iterative conversion with an explicit stack

Any recursive algorithm can be rewritten using a manual stack (an array) and a loop. The idea is to simulate what the call stack does automatically, but in heap memory — which is orders of magnitude larger. 

function drawTreeIterative(x0, y0, len0, angle0, maxDepth) {
  // Each item on our manual stack mirrors a recursive call's arguments
  const stack = [{ x: x0, y: y0, len: len0, angle: angle0, depth: maxDepth }];

  while (stack.length > 0) {
    const { x, y, len, angle, depth } = stack.pop();
    if (depth === 0) continue;

    const x2 = x + Math.cos(angle) * len;
    const y2 = y + Math.sin(angle) * len;
    ctx.beginPath(); ctx.moveTo(x, y); ctx.lineTo(x2, y2); ctx.stroke();

    // Push children — order reversed so left is processed first
    stack.push({ x: x2, y: y2, len: len*0.7, angle: angle+0.436, depth: depth-1 });
    stack.push({ x: x2, y: y2, len: len*0.7, angle: angle-0.436, depth: depth-1 });
  }
}

This version can safely handle depths of 50, 100, or more — the only limit is available heap memory, not the call stack.

2 — Tail-call optimisation (TCO)

When a recursive call is the very last operation in a function, some engines can reuse the current stack frame instead of pushing a new one. This is called tail-call optimisation. Unfortunately, our tree function is not tail-recursive because it makes two recursive calls and must return from the first before making the second. TCO also has limited browser support (only Safari as of 2024).

3 — Trampolining

A trampoline is a loop that repeatedly calls a function that returns either a result or a thunk (a zero-argument function representing the next step). This converts deep recursion into a flat loop without rewriting the entire algorithm. 

const trampoline = fn => (...args) => {
  let result = fn(...args);
  while (typeof result === 'function') result = result();
  return result;
};

4 — Increase the stack size (server-side)

In Node.js you can launch the process with node --stack-size=65536 app.js to enlarge the call stack. This is sometimes appropriate for deeply recursive data-processing scripts but is not available in browsers.

5 — Memoisation / dynamic programming

When recursive calls repeat the same sub-problems (e.g. Fibonacci, shortest paths), caching results in a map eliminates redundant calls and often dramatically reduces depth as well as time complexity.
