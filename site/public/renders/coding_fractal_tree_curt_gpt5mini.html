Recursive Fractal Tree

About recursion depth, stack overflow, and practical alternatives

This fractal tree is drawn using a recursive function that draws a trunk, then recursively draws two (or more) branches rotated and scaled relative to the parent branch. The slider above controls the recursion depth (number of levels). Increasing the depth increases the number of function calls exponentially and produces more detailed trees.

Why depth is limited

Each recursive call consumes a stack frame on the JavaScript call stack. Typical browsers and JavaScript engines limit the maximum call stack size. Very deep recursion (thousands of nested calls) will exceed that limit and throw a stack overflow error (usually seen as "RangeError: Maximum call stack size exceeded").

Practical strategies to avoid stack overflow

  • Keep depth modest: For fractals where branching factor > 1, the number of calls grows exponentially, so a depth of 10–15 is often plenty.
  • Use iterative approaches: Convert recursion to an explicit stack (array) and process frames in a loop. That uses heap memory instead of the call stack and avoids deep call stacks.
  • Use generators or trampolines: Break large recursive tasks into smaller steps and resume via the event loop, or use a trampoline function to avoid deep native recursion.
  • Tail call optimization (TCO): If the language/runtime implements proper TCO and your function is tail-recursive, the runtime can reuse stack frames. Currently, JavaScript engines do not reliably provide TCO in strict mode across browsers, so it is not a dependable solution for production code.
  • Limit branching early: Reduce the branching factor or prune branches based on size or other heuristics (e.g., stop when branch length is below a threshold).

Example: converting recursion to an iterative approach would mean pushing the initial trunk state onto an array, then repeatedly popping a state, drawing it, and pushing its child branches onto the array. That approach avoids relying on the language call stack and gives you explicit control of memory and processing order.
