Widget Dashboard Memory Leak Demonstration

Two side-by-side dashboards. The left intentionally leaks by keeping references and listeners after removal. The right cleans up properly. Use the buttons to add/remove widgets repeatedly and watch the counters.

Leaking version

Visible widgets: 0 • Tracked alive objects: 0
This version attaches listeners and stores references on a global "leakedStore" and never removes them. The GC is not "broken" — it's the code keeping things reachable.

Fixed version

Visible widgets: 0 • Tracked alive objects: 0
This version removes listeners and clears references when widgets are removed so the GC can reclaim memory.
