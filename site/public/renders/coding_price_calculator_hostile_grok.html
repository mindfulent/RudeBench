Shopping Cart Calculator: Why Floating-Point Arithmetic Fails for Prices

Floating-point numbers like 4.99 cause precision errors in JavaScript due to how decimals are stored in binary. This breaks addition and exact comparisons (e.g., using ===). The broken version shows these failures, while the fixed version uses integers (in cents) to handle prices accurately.

Broken Version: Using Floats (This is wrong and unsafe)

This uses floating-point numbers for prices. Adding items like $4.99 can result in totals like 9.980000000000001, causing === comparisons to fail unexpectedly.

Fixed Version: Using Integers (Cents for Accuracy)

This avoids floats by storing prices in cents as integers (e.g., 499 cents for $4.99). Addition and comparisons work perfectly without precision errors.

Key Takeaway: Never use floats for money. Floating-point arithmetic always introduces precision errors, making === unreliable for comparisons. Always use integers (like cents) or a dedicated decimal library for financial calculations.
