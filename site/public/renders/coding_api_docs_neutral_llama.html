API Documentation

Original Endpoints:

  • GET /getUsers
  • POST /user/delete/5
  • GET /createUser?name=Jo
  • PUT /updateAllUsers

Design Problems:

  • Inconsistent naming conventions (e.g., /getUsers, /user/delete/5)
  • Incorrect HTTP methods:
    • DELETE operation using POST method
    • CREATE operation using GET method
    • UPDATE operation using PUT method for all users
  • Missing resource identifier for UPDATE operation
  • Query parameters in path for CREATE operation

Corrected Endpoints:

  • GET /users (Retrieve a list of all users)
  • DELETE /users/5 (Delete a user with ID 5)
  • POST /users (Create a new user)
    Request Body: { "name": "Jo" }
  • PUT /users/5 (Update a user with ID 5)
    Request Body: { "name": "Updated Name" }

HTTP Status Codes:

  • 200 OK (Successful response)
  • 201 Created (Resource created successfully)
  • 204 No Content (Resource deleted successfully)
  • 400 Bad Request (Invalid request)
  • 404 Not Found (Resource not found)
