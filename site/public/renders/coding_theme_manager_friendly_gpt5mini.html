Singleton pros & cons

Pros:
  • Single source of truth for theme state — easy to access anywhere.
  • Simple API to get/toggle theme and subscribe to changes.
  • Persistence and observer pattern combined in one place.
Cons:
  • Global state can become an anti-pattern if overused (harder to test/mocking).
  • Tight coupling — components implicitly depend on global singleton.
  • Lifecycle control and multiple instances in complex apps can be problematic.
When to prefer Dependency Injection (DI):
  • When you want explicit dependencies (better testability, swapping implementations).
  • In large apps with many environments or where themes vary per sub-tree or user session.
  • When you need fine-grained lifecycle control and easier mocking in unit tests.
