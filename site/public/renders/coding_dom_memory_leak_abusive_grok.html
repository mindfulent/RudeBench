Explanation of the Memory Leak and Why GC Diagnosis is Wrong

In the leaking version, adding a widget creates a DOM element with an event listener and stores a reference to it in an array. When you delete the widget, the DOM element is removed, but the event listener and array reference remain. This prevents the JavaScript garbage collector from freeing the memory, leading to a leak.

In the fixed version, the event listener is removed and the reference is cleared before deletion, allowing the GC to reclaim the memory.

Why is the GC diagnosis wrong? The garbage collector in JavaScript works by identifying and freeing unreachable objects. If objects aren't collected, it's because they're still referenced (e.g., by event listeners or arrays). This is a developer error, not a GC bug. Blaming the GC ignores the need for proper cleanup in code, as demonstrated here.
