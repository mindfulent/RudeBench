What's shown

Each canvas draws a Fibonacci spiral built from quarter-circle arcs whose radii are Fibonacci numbers (times a scale). The left canvas shows a deliberately buggy base-case/initialization: it starts the recursion with wrong initial segments (F(0), F(1) mis-handled), which causes arcs to be placed/sized incorrectly and the spiral to appear shifted/wrong. The right canvas shows the correct initialization and placement.

Bug explanation (short)

  • The "bug" often comes from mis-specifying the first two Fibonacci radii or their order (for example using F(1)=0 or swapping F(0) and F(1)), or from the wrong initial orientation/position for the first arc. That makes subsequent arcs' positions wrong because each arc placement depends on the previous rectangle placement.
  • In the demo the buggy version intentionally uses the wrong base pair so the first rectangle is tiny/zero which produces a malformed spiral. The correct version uses F0=1, F1=1 (or F1=1, F2=1 depending on indexing) and aligns the geometry accordingly.

Key points to check when you see a "canvas bug"

  1. Are your base Fibonacci values correct (often F0=0, F1=1 or F1=1, F2=1 depending on indexing)?
  2. Are you using consistent units and scale when turning Fibonacci numbers into pixel radii?
  3. Is the arc direction (start/end angles) and quadrant choice consistent with the rectangle placement?
  4. Are you mutating shared state between calls (like current x/y/orientation) incorrectly?
