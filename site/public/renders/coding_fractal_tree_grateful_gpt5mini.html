Explanation — recursion depth and practical notes

Recursion depth: The slider controls how many recursive levels the draw function runs. Depth = 1 draws just the initial trunk; larger depths produce more branches exponentially.

Why limits matter: Each recursive call consumes a stack frame. Deep recursion may exhaust the JavaScript call stack and throw a "Maximum call stack size exceeded" error. Browsers typically allow thousands of nested calls, but recursion that grows exponentially (fractal trees) can hit practical performance limits long before the stack does, due to many function calls and drawing operations.

Practical solutions:

  • Lower the depth (this demo caps depth at 12 for responsiveness).
  • Use an iterative approach (stack/queue) to convert recursion into a loop, avoiding deep call stacks.
  • Throttle or chunk drawing over multiple animation frames (requestAnimationFrame) to keep UI responsive for large trees.
  • Memoize or reuse path geometry where possible, or render to an offscreen canvas and reuse the bitmap.
