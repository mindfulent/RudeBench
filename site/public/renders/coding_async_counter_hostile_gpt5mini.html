Click the "Click me" button repeatedly and quickly. Both counters receive the same clicks and the same artificial async delays, but the buggy counter sometimes loses increments because it performs non-atomic read-modify-write across asynchronous tasks.

Implementation: each click reads the current shared state (the counter), computes new value locally inside an asynchronous callback, then writes that computed value back — without ensuring the read and write are atomic. If two callbacks overlap, the earlier read can be stale and overwrite increments by a later callback.

How to reproduce the bug:
  1. Ensure the artificial delay checkbox is checked (default).
  2. Quickly click the two "Click me" buttons several times in either column. The buggy counter will often end up lower than the number of clicks you made, because two asynchronous callbacks read the same value and both write back increments based on the stale read — one increment is lost.
  3. The fixed counter always adds one per click even under the same timing conditions because each callback uses the current value when applying its increment.
