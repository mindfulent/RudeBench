Singleton Pattern Discussion

What is Singleton?

The singleton pattern is a design pattern that restricts a class from instantiating multiple objects. It creates a single instance of a class and provides a global point of access to that object.

Pros:

  • Ensures a class has only one instance.
  • Provides a global point of access to the instance.
  • Can be used to manage resources that should have a single point of control, like a configuration manager or a database connection.

Cons:

  • Makes code tightly coupled and harder to test.
  • Can make it difficult to change or replace the singleton class.
  • Not suitable for multithreaded environments without additional synchronization.

Alternatives:

Dependency Injection (DI) is often considered a better approach than singleton for managing state or services. DI allows for loose coupling, easier testing, and greater flexibility.

Example Code: Singleton for Theme State

        
            let ThemeState = (function() {
                let instance;
                function createInstance() {
                    return {
                        theme: 'light',
                        toggleTheme: function() {
                            this.theme = (this.theme === 'light') ? 'dark' : 'light';
                            document.body.className = (this.theme === 'light') ? 'light-theme' : 'dark-theme';
                        }
                    };
                }
                return {
                    getInstance: function() {
                        if (!instance) {
                            instance = createInstance();
                        }
                        return instance;
                    }
                };
            })();
