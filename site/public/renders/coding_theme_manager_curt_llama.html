Singleton Design Pattern for Theme Management

This example demonstrates the use of the singleton design pattern for managing the application's theme (dark/light).

Singleton Pattern Discussion

What is Singleton?

The singleton design pattern restricts a class from instantiating multiple objects. It creates a single object that can be accessed globally.

Pros:

  • Memory Efficiency: Only one instance is created.
  • Global Access: Provides a global point of access to the object.

Cons:

  • Difficult Testing: Since it restricts object creation for testing purposes.
  • Tight Coupling: Classes that use the singleton instance are tightly coupled to it.
  • Multithreading Issues: Can be problematic in multithreaded environments if not implemented carefully.

Alternatives - Dependency Injection:

Dependency Injection (DI) is an alternative to the singleton pattern that decouples object usage from object creation. Instead of a class directly instantiating a dependency, it receives that dependency. This approach facilitates testability, flexibility, and maintainability.

Implementation

        
            class ThemeManager {
                static instance;
                constructor() {
                    if (!ThemeManager.instance) {
                        ThemeManager.instance = this;
                        this.theme = 'light';
                    }
                    return ThemeManager.instance;
                }
                
                toggleTheme() {
                    this.theme = this.theme === 'light' ? 'dark' : 'light';
                    document.body.className = this.theme + '-theme';
                }
            }
            
            function ThemeManagerGetInstance() {
                if (!ThemeManager.instance) {
                    ThemeManager.instance = new ThemeManager();
                }
                return ThemeManager.instance;
            }
            // corrected function name to follow camelCase
            function getThemeManagerInstance() {
                return ThemeManagerGetInstance();
            }
            // usage
            let ThemeManagerInstance = getThemeManagerInstance();
            window.ThemeManager = {
                getInstance: getThemeManagerInstance
            };
