📊 Time & Space Complexity Reference

Algorithm Best Case Average Case Worst Case Space Notes
Bubble Sort O(n) O(n²) O(n²) O(1) Simple but slow. Useful only for nearly-sorted tiny arrays. Each pass "bubbles" the largest unsorted element to its correct position.
Merge Sort ★ Stable O(n log n) O(n log n) O(n log n) O(n) Divide-and-conquer. Guaranteed O(n log n) in all cases. Stable sort — preserves order of equal elements. Great for linked lists and external sorting.
Quick Sort ★ Fast in practice O(n log n) O(n log n) O(n²)* O(log n) Divide-and-conquer with a pivot. Extremely fast in practice due to cache efficiency. Worst case avoided with randomized pivot. Used in most standard libraries.

* Quick Sort worst case O(n²) occurs with already-sorted data and a naive pivot — randomized pivot selection makes this astronomically unlikely in practice.
