REST Endpoint Documentation

Original Endpoints

GET /getUsers

Retrieves a list of users

Problem: Inconsistent naming convention

Corrected: GET /users

POST /user/delete/5

Deletes a user with ID 5

Problem: Incorrect HTTP method, inconsistent naming convention, and action embedded in path

Corrected: DELETE /users/5

GET /createUser?name=Jo

Creates a new user with name "Jo"

Problem: Incorrect HTTP method, inconsistent naming convention, and query parameter for creation

Corrected: POST /users with JSON payload

PUT /updateAllUsers

Updates all users

Problem: Incorrect HTTP method, non-standard use case, and missing resource identification

Corrected: PUT /users/5 or PATCH /users

Corrected Endpoints

GET /users

Retrieves a list of users

HTTP Method: GET

Status Code: 200 OK

DELETE /users/5

Deletes a user with ID 5

HTTP Method: DELETE

Status Code: 204 No Content

POST /users

Creates a new user

HTTP Method: POST

Request Body: JSON payload (e.g., { "name": "Jo" })

Status Code: 201 Created

PUT /users/5

Updates a user with ID 5

HTTP Method: PUT

Request Body: JSON payload (e.g., { "name": "Updated Name" })

Status Code: 200 OK

PATCH /users

Partially updates all users

HTTP Method: PATCH

Request Body: JSON payload (e.g., { "name": "Updated Name" })

Status Code: 200 OK
