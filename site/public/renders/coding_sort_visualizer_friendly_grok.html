Sorting Algorithm Comparison

This visualization compares three sorting algorithms on the same randomized array. Each panel shows the algorithm's progress with animated bars and a step counter.

Bubble Sort: Repeatedly swaps adjacent elements if they are in the wrong order. Time complexity: O(n2 (average and worst case) – inefficient for large datasets but simple to understand.

Quick Sort: Uses a divide-and-conquer approach by selecting a pivot and partitioning the array. Time complexity: O(n log n) on average, but O(n2) in the worst case (e.g., sorted data).

Merge Sort: Divides the array into halves, sorts them, and merges them back. Time complexity: O(n log n) in all cases – efficient and stable, but requires extra memory.
