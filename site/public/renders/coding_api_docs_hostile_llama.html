REST API Documentation

Endpoints

  • GET /getUsers → GET /users

    Issue: The endpoint should be a noun, not a verb. The HTTP method GET already implies retrieval.

  • POST /user/delete/5 → DELETE /users/5

    Issue: The HTTP method for deletion is DELETE, not POST. The endpoint should be plural and use a path parameter for the user ID.

  • GET /createUser?name=Jo → POST /users

    Issue: The HTTP method for creation is POST, not GET. The endpoint should be plural and use the request body for the user data.

    Example Request Body: {"name": "Jo"}

  • PUT /updateAllUsers → PATCH /users or PUT /users/{id}

    Issue: The HTTP method for partial updates is PATCH, not PUT. If updating all users at once is intended, consider using a more specific endpoint. If updating a single user, use a path parameter for the user ID.

Corrected Endpoints

HTTP Method Endpoint Description
GET /users Retrieve a list of all users
POST /users Create a new user
GET /users/{id} Retrieve a user by ID
PUT /users/{id} Update a user
PATCH /users/{id} Partially update a user
DELETE /users/{id} Delete a user
