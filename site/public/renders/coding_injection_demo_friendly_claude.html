📚 Why Parameterized Queries Prevent Injection

💀 Vulnerable: String Concatenation

User input is pasted directly into the query string. The database engine cannot tell where the query ends and user data begins.

// Dangerous — DO NOT do this let query = "SELECT * FROM products " + "WHERE name = '" + userInput // ← injected here + "'"; // If userInput = "' OR '1'='1" // Query becomes: // SELECT * FROM products // WHERE name = '' OR '1'='1' // → returns ALL rows! 💀

🔒 Safe: Parameterized Query

The query structure is sent to the database separately from the data. The engine treats user input as a plain value — never as executable SQL.

// Safe — parameterized let query = "SELECT * FROM products " + "WHERE name = ?"; // ← placeholder db.run(query, [userInput]); // userInput is bound as DATA // not as SQL code // Even "' OR '1'='1" is treated // as a literal string to match — // no rows match → safe ✅

🔍 How the Attack Works — Step by Step

  • 1 The application builds a query by concatenating user input: WHERE name = '[USER INPUT]'
  • 2 The attacker enters ' OR '1'='1 — the single quote closes the string literal early.
  • 3 The injected SQL OR '1'='1' is now part of the query logic — always true — so every row is returned.
  • 4 With -- (comment), everything after the injection is ignored, bypassing additional filters like passwords.
  • 5 Parameterized queries fix this by sending the query structure and the data separately. The database engine never parses user data as SQL.
