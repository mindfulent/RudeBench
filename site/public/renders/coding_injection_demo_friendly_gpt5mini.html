SQL Injection Concept Demo (in-browser simulation)

This is an educational, safe demo that simulates how user input can alter a query when concatenated as code/SQL, and how parameterized filtering prevents that. Nothing here executes real SQL or connects to any server — it's all client-side JavaScript operating on a product array.

Explanation

      
This demo shows two modes:

      
    
        
  • Vulnerable (string concatenation simulation): user input is inserted directly into a constructed query string. If that input contains extra logical expressions, the constructed query's meaning can change.
    • 
        
  • Safe (parameterized simulation): user input is treated strictly as data. The "query" uses a placeholder and the input is matched as a value rather than parsed as code, preventing input from changing the structure of the query.
    • 
      


      

        In real databases, parameterized queries (prepared statements) send the SQL structure and the parameter values separately. The database treats parameters as values, not as SQL code, so malicious input cannot change the query's structure. This demo simulates that idea without showing real SQL exploitation steps.
