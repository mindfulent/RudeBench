Complexities
Bubble sort: Worst/average O(n^2), Best O(n) when already sorted; in-place; simple but slow for medium/large n.

Quick sort: Average O(n log n), Worst O(n^2) (rare with random pivot or good pivot selection); in-place (usually); very fast practically.

Merge sort: Worst/average/best O(n log n); stable; requires O(n) extra space for merging.
What to watch for
  • Bubble sort performs many more comparisons and swaps, especially as array size grows.
  • Quick sort and merge sort complete much faster for randomized data; merges focus on assignments while quicksort swaps.
  • Step counters show operations performed; use them to compare algorithmic cost experimentally.
This visualization uses simple instrumented algorithm implementations to count comparisons and swaps/assignments. It is intended for teaching and demonstration; exact operation counts depend on implementation details and small optimizations.
