Fibonacci Spiral Demonstration

This page demonstrates two versions of a Fibonacci spiral animation on a canvas. One version contains a common calculation bug, while the other is fixed. This illustrates that perceived rendering issues are often due to underlying logic rather than the canvas API itself.

Buggy Fibonacci Spiral

This version has a common bug in how the spiral segments are calculated or connected, leading to a disconnected or misaligned appearance. The canvas API is *not* at fault here.

Fixed Fibonacci Spiral

This version correctly calculates and connects the spiral segments, resulting in a smooth, continuous Fibonacci spiral. This demonstrates the canvas API functions as expected when provided with correct data.

Explanation of the Bug and Fix

The common bug in drawing a Fibonacci spiral using arcs is often related to the *starting point and angle* for each subsequent arc. A Fibonacci spiral is constructed by drawing quarter-circle arcs in squares whose side lengths correspond to Fibonacci numbers. Each new arc must start exactly where the previous one ended, and its center must be correctly offset from the previous arc's center based on the growing square.

Common Bug: Often, developers might correctly calculate the Fibonacci sequence and the arc radii, but fail to accurately update the `x`, `y` coordinates for the arc's center and the `startAngle`, `endAngle` for each successive arc. For instance, they might keep the center fixed, or increment it incorrectly, causing a visual "jump" or disconnect between segments. Another common mistake is not correctly rotating the drawing direction for each quadrant.

The Fix: The fix involves meticulously tracking the current drawing position (the "corner" of the current square), the center of the current arc, and the correct start and end angles for the arc, ensuring that each arc seamlessly continues from the previous one. This requires careful management of state (position, direction) as the spiral unfolds.

In the code above, the "buggy" version might simplify the center calculation or angle rotation, causing the disconnect. The "fixed" version precisely updates these parameters for each segment.
