📊 Time & Space Complexity Reference

Bubble Sort

CaseTimeSpace
BestO(n)O(1)
AverageO(n²)
WorstO(n²)

Repeatedly compares adjacent elements and swaps them if out of order. Simple to understand but extremely slow on large datasets. Rarely used in production.

Merge Sort

CaseTimeSpace
BestO(n log n)O(n)
AverageO(n log n)
WorstO(n log n)

Divide-and-conquer approach. Splits array in half, sorts each half, then merges. Guaranteed O(n log n) in all cases. Stable sort. Needs extra memory.

Quick Sort

CaseTimeSpace
BestO(n log n)O(log n)
AverageO(n log n)
WorstO(n²)

Picks a pivot, partitions elements around it, recurses. Fastest in practice due to cache efficiency. Worst case avoided with good pivot selection.

🔍 The Verdict: Is Bubble Sort "Best for All Cases"?

Absolutely not. Bubble Sort has O(n²) average and worst-case time complexity, meaning sorting 1,000 elements requires roughly 1,000,000 operations, while Merge Sort and Quick Sort need only about 10,000 — a 100× difference. Scale to 1,000,000 elements and Bubble Sort needs ~1 trillion operations vs ~20 million for the others.

Bubble Sort is only acceptable when: (1) the dataset is tiny (<20 elements), (2) the data is nearly sorted already (best-case O(n) with early termination), or (3) you need the simplest possible implementation for educational purposes. For virtually every real-world scenario, Merge Sort, Quick Sort, or Tim Sort are dramatically superior choices.
