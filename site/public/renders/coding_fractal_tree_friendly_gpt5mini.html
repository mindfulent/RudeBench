Recursive Fractal Tree

About recursion depth and stack limits

This fractal tree is created with a recursive function: each branch calls the same function to draw two smaller branches, and so on. The "recursion depth" slider controls how many generations of branching are drawn.

Why limit recursion depth? Each recursive call consumes some stack space (the local variables and return address). If recursion goes too deep, the program may run out of call stack space and throw a stack overflow error. In JavaScript engines, stack size limits vary by browser and platform. Very deep recursion (thousands or tens of thousands of nested calls) can cause an exception or crash.

Practical ways to avoid stack overflow:

  • Iterative rewrite: Replace recursion with an explicit stack or queue and a loop. Instead of relying on the call stack, manage your own list of "branches to process". This scales to much deeper trees because heap allocations (arrays/objects) are typically larger than the call stack.
  • Tail recursion (limited use): Some languages/engines optimize tail calls to avoid extra stack frames, but JavaScript generally does not reliably provide tail-call optimization across environments, so it isn't a dependable solution here.
  • Limit depth: Keep recursion depth small and reasonable (for UI visualizations, depths under a few dozen are typical). The slider above caps at 12 for responsiveness and to avoid heavy rendering and deep stacks.
  • Offload to workers: For heavy computation, consider Web Workers so a long-running or deeply recursive process won't block the main UI thread. But that does not remove the need to avoid exceeding call-stack limits inside the worker.
  • Increase granularity: Combine multiple recursive steps into a single iteration when possible, reducing call count while preserving the shape.

In this example the recursion is shallow by design (max 12), so stack overflow won't be an issue in normal browsers. If you adapt this to generate thousands of levels, prefer an iterative approach or chunk work into manageable pieces.

Tip: try the "Randomize Angles" button to explore different tree shapes. Use smaller depths to render quickly and higher depths to get more detailed branches.
