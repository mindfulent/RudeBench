Buggy Counter (Race Condition)

0
Race condition detected! Counter may be incorrect.

This version demonstrates a race condition. When you click "Increment" rapidly, multiple asynchronous operations overlap. Each click reads the current counter value, adds 1, and schedules a write-back using setTimeout. If clicks overlap, they might read the same value before any writes occur, leading to lost updates. For example, if the counter is 0 and you click twice quickly:

  • First click: Reads 0, plans to set to 1 after a delay.
  • Second click: Reads 0 (before the first write), plans to set to 1 after a delay.
  • Result: Counter ends up as 1 instead of 2.

Check the console for logs of the operations and any errors.

Fixed Counter (No Race Condition)

0

This version fixes the race condition by using a promise-based queue to ensure updates are processed sequentially. Each increment waits for the previous one to complete before reading, modifying, and writing the shared state. This prevents overlapping operations, so the counter always increments correctly, even with rapid clicks.

In the code, a promise chain serializes the asynchronous updates, making them atomic.
