Dashboard Widget Memory Leak Demo

Two side-by-side dashboards: left intentionally leaks (attaches listeners and keeps references on removal), right is fixed (cleans up). Use "Add widget" and "Remove widget" to create churn. Watch the tracked counters below.

Explanation: The left example demonstrates a common pattern that causes objects to be retained even after DOM nodes are removed: storing references to DOM nodes (or state) in long-lived structures and attaching closures/event listeners that close over those nodes. The garbage collector cannot free memory while reachable references remain. The dashboard widget's growing memory usage is therefore not a GC bug but a retention bug in application code. Use explicit cleanup (remove listeners, null references) like the fixed example.
