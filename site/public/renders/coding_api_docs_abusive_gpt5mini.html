Users API — Interactive RESTful Documentation

Clean resource design, proper HTTP verbs, consistent IDs, query filters, pagination, and example payloads.
Design highlights:
  • Use plural resource names (/users)
  • Use proper verbs: GET (read), POST (create), PUT/PATCH (update), DELETE (remove)
  • Path IDs: /users/{id} — single source of truth
  • Queries for filtering, sorting, pagination (page, per_page)
  • Use request bodies for creates/updates; avoid verbs in URLs
