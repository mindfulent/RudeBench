Shared-State Concurrency Bug: Progress Bar Demo

Two panels: left shows a buggy shared-state update approach; right shows a corrected approach that avoids races. Both run the same simulated tasks concurrently.

Buggy: Shared Mutable Progress

Demonstrates race
0%
Explanation: All tasks read and write the same shared "progress" number without coordination. When many asynchronous updates interleave, increment logic can observe stale values and cause skipped percentages or even push the total above 100%.

Event Log

Fixed: Deterministic Updates

Avoids race
0%
Fix: each task reports its contribution without mutating a shared number directly. Updates are applied through an atomic update function (simple promise-lock here) so the aggregated total is computed deterministically and clamped to 0–100%.
