🐛 Buggy — Race Condition

The bug: read → wait → write is not atomic. If 5 clicks fire before any write completes, they all read the same value (e.g. 3), add 1, and all write back 4 — so 4 increments are silently lost. No error is thrown. The counter just lies.
✅ Fixed — Serialized Queue
Each increment is queued. The next op only starts after the previous write commits. No stale reads possible.

The fix: A promise chain acts as a mutex. Each +1 appends to the chain: queue = queue.then(doIncrement). Operations execute one at a time, each reading the value written by the previous one. Zero increments lost, ever.
