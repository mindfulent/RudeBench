Concurrency Bug Demonstration in Progress Bar

This page shows a buggy and corrected implementation of a progress bar updated by multiple concurrent tasks. The buggy version demonstrates race conditions with shared state, while the corrected version fixes it.

Buggy Implementation

In this version, multiple tasks run concurrently using setTimeout and Promises. They access and update a shared progress variable without synchronization. This leads to race conditions: tasks may read the same stale value, causing the progress to skip percentages, show incorrect values, or even exceed 100% (e.g., if updates overwrite each other based on outdated states).

Corrected Implementation

In this version, tasks are executed sequentially using Promises and async/await. This ensures that each task waits for the previous one to complete before updating the shared progress variable. The fix prevents race conditions by maintaining a consistent state, so the progress bar updates accurately without skipping values or exceeding 100%.
