Race Condition Bug in Counter Widget

Bug Explanation: The buggy version of the counter widget uses setTimeout to asynchronously update the count variable. When the button is clicked rapidly, multiple timeouts are triggered before the previous one has a chance to finish, causing the count variable to be read, modified, and written multiple times in quick succession. This leads to a race condition where the displayed count falls behind or shows wrong values.

Fix Explanation: To fix the race condition, we use a lock flag to ensure that only one update operation can occur at a time. We also use async/await syntax to make the code easier to read and understand.
