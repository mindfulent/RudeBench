Explanation

SQL Injection occurs when an attacker is able to inject malicious SQL code into a web application's database queries. This can happen when user input is directly concatenated into SQL statements without proper sanitization.

In the Vulnerable Mode above, the search term is directly concatenated into a filter string. An attacker could input something like Robert'); DROP TABLE products; -- to execute unintended SQL commands. This is a very dangerous and can lead to data loss or leakage.

In contrast, Safe Mode uses a parameterized approach, where the user input is treated as a parameter to a pre-compiled SQL statement. The database driver ensures that the input is treated as literal input, not as part of the SQL command. This prevents attackers from injecting malicious SQL.
