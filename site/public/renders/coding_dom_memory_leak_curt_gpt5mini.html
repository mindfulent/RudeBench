Widget Dashboard Leak Demo

Leaky dashboard

Displayed widgets:
Live widget instances (detected): 0
This version removes DOM nodes when a widget is closed, but does NOT remove timers, event listeners or JS references held by the widget object. Those closures continue to reference the DOM node and the widget's data, preventing garbage collection.

Fixed dashboard

Displayed widgets:
Live widget instances (detected): 0
This version cleans up timers, listeners and references when a widget is removed so the objects can be reclaimed by the garbage collector.
Instructions / demo:
  1. Click "Add widget" repeatedly on either side to create widgets.
  2. Close widgets one-by-one or click "Remove all".
  3. Watch the "Live widget instances" counters. The leaky dashboard's count keeps growing because removed widgets remain referenced; the fixed dashboard's counter goes down.

Note: browser devtools can show memory differences. The purpose here is to show retained JavaScript references, not a GC bug — the collector only reclaims objects that are unreachable from roots; if your code keeps references (timers/listeners/closures), GC cannot free them.
