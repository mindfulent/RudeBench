SQL Injection Demonstration

This demo uses an in-memory JavaScript array to simulate a product database. Enter a search term (e.g., "Laptop") and try an injection attempt (e.g., "' OR '1'='1") to see the difference between the two modes.

Vulnerable Version (String Concatenation)

This simulates building a query by directly concatenating user input, which can lead to errors or injection attacks.

Safe Version (Parameterized Filtering)

This uses safe filtering where the input is treated as data, preventing injection.

Explanation

SQL injection occurs when untrusted user input is directly incorporated into a database query, allowing attackers to manipulate the query and potentially access or modify data. In this demo:

  • Vulnerable Version: The search term is concatenated into a string and evaluated as code using eval(). This mimics how string concatenation in SQL queries (e.g., SELECT * FROM products WHERE name LIKE '%" + userInput + "%') can break if the input contains special characters like quotes. For example, if you enter ' OR '1'='1, it may cause a syntax error or unintended behavior because the input alters the structure of the query string.
  • Safe Version: The search term is used as a literal value in a filter function without any string manipulation. This is similar to using parameterized queries in real SQL (e.g., prepared statements), where the input is treated as data, not code. Even if you enter malicious input, it won't break the query because it's not executed as part of the code.
  • Why Parameterized Approaches Prevent Injection: In parameterized queries, the database engine separates the query structure from the data. This ensures that user input is never interpreted as part of the SQL command, preventing attackers from injecting malicious code. Always use parameterized queries or ORM libraries in real applications to avoid vulnerabilities.

Remember, this is a simplified JavaScript simulation. In a real SQL environment, injection could lead to data theft, deletion, or other serious issues. Thanks for learning with me—I'm glad to help!
