SQL Injection Demonstration

This demo uses an in-memory JavaScript array as a product database. Enter a search term and select a mode to see how inputs are handled.



Explanation of SQL Injection and Parameterized Queries

What is SQL Injection? SQL injection is a security vulnerability that occurs when user input is directly incorporated into SQL queries without proper sanitization. An attacker can manipulate the input to alter the query's intent, potentially accessing, modifying, or deleting data.

How This Demo Works:

  • Vulnerable Mode: The search term is directly used to construct a regular expression (e.g., new RegExp(searchTerm, 'i')). This mimics string concatenation in SQL queries. If you enter something like ".*", it will match all products, demonstrating how an attacker could bypass filters (similar to injecting "OR 1=1" in SQL). If the input is invalid (e.g., malformed regex), it may result in an error.
  • Safe Mode: The search term is treated as a literal string using a simple inclusion check (e.g., product.name.toLowerCase().includes(searchTerm.toLowerCase())). This is analogous to using parameterized queries in SQL, where user input is isolated as data, not part of the query logic.

Why Parameterized Approaches Prevent Injection: In parameterized queries (or prepared statements), user input is sent separately from the query structure. For example, in SQL, you might use: SELECT * FROM products WHERE name LIKE ?, where "?" is a placeholder for the user's input. The database engine treats the input as data, not code, preventing it from altering the query. This demo's safe mode isolates the input similarly, ensuring it can't manipulate the filtering logic. Always use parameterized queries in real applications to avoid SQL injection vulnerabilities.

Try It Out:

  • Search for "Lap" in both modes – it should work as expected.
  • In Vulnerable Mode, try ".*" to see all products (injection simulation).
  • In Safe Mode, ".*" will be treated literally and likely return no results.
