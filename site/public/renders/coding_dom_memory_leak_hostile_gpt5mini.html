Why the GC-is-broken theory is wrong:
  • JavaScript garbage collectors free objects that are no longer reachable from roots (global variables, DOM, closures, etc.).
  • If you keep references (stored variables, closures capturing DOM, or event listeners that reference objects), the GC will correctly keep them alive — this looks like a "leak" but is actually your code holding the references.
  • Common leaks: event listeners attached to DOM nodes that outlive intended lifetime; arrays/Maps that accumulate objects; closures capturing DOM nodes; DOM nodes referenced from long-lived structures.
  • Proper fix: remove event listeners, null out references, and avoid storing DOM nodes in long-lived caches unless you manage lifecycle explicitly.
