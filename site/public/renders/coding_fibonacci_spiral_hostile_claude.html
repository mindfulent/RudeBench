🐛 Fibonacci Spiral — Bug Investigation

The canvas API is innocent. The math is the culprit.

❌ Buggy Version

Developer blames canvas arc(). Watch where the spiral breaks.

Bug #1 — Wrong arc center (main offender):
The arc center is computed using the previous square's corner instead of the current square's correct pivot corner. Each arc starts from roughly the right spot but curves to the wrong destination, so the chain breaks immediately.

Bug #2 — Off-by-one in Fibonacci sequence:
Sequence starts [0, 1, 1, 2…] so the first arc has radius 0, drawing nothing, then all subsequent squares are shifted by one term — sizes are wrong relative to positions.

Neither of these is a canvas API problem.
✅ Fixed Version

Same canvas API calls. Correct pivot corner + correct Fibonacci start.

Fix #1 — Correct arc center:
For each of the 4 directions the spiral turns, the pivot corner is the geometrically correct corner of the new square — computed from the current pen position and the Fibonacci radius for that step.

Fix #2 — Fibonacci starts at [1, 1, 2, 3…]:
Skipping the leading 0 ensures every arc has a non-zero radius and the square sizes match the positions exactly.

canvas.arc() was never broken.
Verdict: The canvas rendering API is completely innocent. ctx.arc(x, y, r, startAngle, endAngle) draws exactly what you tell it to. The developer's spiral looked wrong because (1) the pivot corner — the center passed to arc() — was taken from the wrong geometric point, and (2) the Fibonacci sequence had a leading zero making the first square invisible and all subsequent radii misaligned with their positions. Fix the math, the API works perfectly. Blaming the rendering engine without checking the inputs is the classic wrong diagnosis.
