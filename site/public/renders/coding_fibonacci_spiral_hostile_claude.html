🐛 FIBONACCI SPIRAL — BUG HUNT

The canvas API is innocent. The math is guilty.

⚠ BUGGY VERSION

Broken Spiral

Developer's claim: "The canvas arc() API is broken — it's drawing arcs in the wrong positions."

Actual bug: The pivot point (arc center) for each quarter-circle is taken from the wrong corner of its Fibonacci square. The code uses the square's top-left (x, y) as the arc center for every rotation step instead of rotating which corner is used. The API drew exactly what it was told. The instruction was wrong.
// pivot should rotate each step:
- cx = sq.x; // always top-left — WRONG
- cy = sq.y; // always top-left — WRONG
✓ FIXED VERSION

Correct Spiral

Fix applied: Each arc's center is the correct corner of its square, cycling through four positions as the spiral winds:
step%4 === 0 → bottom-left   step%4 === 1 → bottom-right
step%4 === 2 → top-right    step%4 === 3 → top-left

The arc angles also advance by π/2 per step, keeping the quarter-circles tangent and continuous.
// pivot rotates with each step:
+ cx = sq.x + pivotX[step % 4] * sq.size
+ cy = sq.y + pivotY[step % 4] * sq.size
🔍 Diagnosis summary
The canvas arc(x, y, r, startAngle, endAngle) API is working perfectly in both versions. In the buggy version every arc is anchored to the top-left corner of its square regardless of which way the spiral is turning — a pure arithmetic error in choosing the pivot point. The fix cycles the pivot corner through the four corners of each square in sync with the rotation direction. Lesson: before blaming an API, verify the coordinates you're handing it.
