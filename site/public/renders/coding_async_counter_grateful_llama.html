Buggy Version

Count: 0

Race Condition Explanation:

When rapidly clicking the button, multiple setTimeout calls are made.

However, these asynchronous updates read the current count, increment it, and then write it back.

If one setTimeout call reads the count before another writes it back, the count can fall behind or show wrong values.

Corrected Version

Count: 0

Fix Explanation:

Instead of reading, modifying, and then writing the shared count variable asynchronously,

the corrected version directly updates the count display using a synchronous operation.

This ensures that each button click updates the count correctly, even with rapid clicks.
