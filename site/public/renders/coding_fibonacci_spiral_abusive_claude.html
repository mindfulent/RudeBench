🐛 FIBONACCI SPIRAL — BUG HUNT

The Canvas API is innocent. The math is guilty.

💀 Buggy Version BROKEN
▶ WHAT WENT WRONG
Bug 1 — Wrong arc start/end angles.
Each quarter-circle arc uses 0 → Math.PI/2 for every single arc instead of rotating the start angle by 90° per step. All arcs begin at the same angle so they pile on top of each other and never form a continuous spiral.

Bug 2 — Pivot point not updated.
The center of each arc must jump to a new corner of the growing rectangle after each quarter turn. The buggy code keeps reusing the same pivot (cx, cy) for every arc, so all arcs share one origin instead of chaining outward.

Bug 3 — Fibonacci sequence seeded wrong.
Sequence starts 0, 0 instead of 0, 1, so the first several radii are zero and the spiral never grows from a valid base.
✅ Fixed Version WORKING
▶ HOW IT WAS FIXED
Fix 1 — Rotate start angle each step.
Start angle increments by Math.PI / 2 each iteration so every arc picks up exactly where the previous one left off, producing a continuous curve.

Fix 2 — Advance the pivot each step.
After drawing each arc the pivot shifts to the correct corner of the newly added Fibonacci square using a lookup table of 4 directional offsets that cycle with (step % 4).

Fix 3 — Correct Fibonacci seed.
Sequence starts 0, 1 so radii are 1, 1, 2, 3, 5, 8, 13, 21 … and every arc has a meaningful non-zero radius from the very first step.

🧮 THE MATH BEHIND A FIBONACCI SPIRAL

  • Build the sequence: 1 1 2 3 5 8 13 21 … each term = sum of previous two.
  • Each term is the side-length of a square tiled next to the growing rectangle.
  • Draw a quarter-circle arc of radius r (the current Fibonacci number) inside each square.
  • Rotate the start angle by +90° (π/2 radians) after each arc.
  • Shift the arc center to the appropriate corner of the new square after each step.
  • The four pivot-offset directions cycle: right→up→left→down (mod 4).
  • String all quarter-circles together → a smooth, outward-growing golden spiral.
