API Documentation

Original Endpoints (Problematic)

  • GET /getUsers
  • POST /user/delete/5
  • GET /createUser?name=Jo
  • PUT /updateAllUsers

Design Problems

  • UnRESTful method usage: Using GET for /getUsers and /createUser, POST for /user/delete/5, and PUT for /updateAllUsers.
  • Inconsistent naming: Using both singular and plural forms for resources (/user and /getUsers).
  • Query parameters in GET requests: Using query parameters to pass data in /createUser?name=Jo.
  • Missing resource identifier: /updateAllUsers does not specify which users to update.
  • Incorrect HTTP method for delete operation: Using POST instead of DELETE for /user/delete/5.

Corrected Endpoints (Following REST best practices)

  • GET /users - Retrieve a list of all users.
  • POST /users - Create a new user. Request Body: { name: "Jo" }
  • GET /users/{id} - Retrieve a specific user by ID.
  • PUT /users/{id} - Update a specific user by ID. Request Body: { name: "Jo Updated" }
  • DELETE /users/{id} - Delete a specific user by ID.

Example Status Codes:

  • 200 OK - Successful request.
  • 201 Created - Resource created successfully.
  • 204 No Content - Resource deleted successfully.
  • 404 Not Found - Resource not found.
  • 400 Bad Request - Invalid request data.
