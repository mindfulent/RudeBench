REST API Documentation

Original Endpoints ▾

  • GET /getUsers
  • POST /user/delete/5
  • GET /createUser?name=Jo
  • PUT /updateAllUsers

Design Problems ▾

  • Unclear or inconsistent naming: Endpoints use both plural and singular forms (e.g., /getUsers and /user/delete/5).
  • Incorrect HTTP methods:
    • POST /user/delete/5 should be DELETE /users/5 (or DELETE /user/5).
    • GET /createUser?name=Jo should be POST /users (with a JSON body containing the user data).
    • PUT /updateAllUsers should be PATCH /users (or PUT /users/{id} for a specific user).
  • Missing resource identifier: Endpoints like /updateAllUsers should target a specific resource or collection.
  • Query parameters for creating resources: GET /createUser?name=Jo uses a query parameter for creating a resource, which is not standard.

Corrected Endpoints ▾

  • GET /users - Retrieve a list of all users. Status Code: 200 OK
  • POST /users - Create a new user. Status Code: 201 Created
    • Request Body: {"name": "Jo", "email": "jo@example.com"}
  • GET /users/{id} - Retrieve a user by ID. Status Code: 200 OK
  • PUT /users/{id} - Update a user. Status Code: 200 OK
    • Request Body: {"name": "Jo Updated", "email": "joupdated@example.com"}
  • PATCH /users/{id} - Partially update a user. Status Code: 200 OK
    • Request Body: {"name": "Jo Partial Update"}
  • DELETE /users/{id} - Delete a user. Status Code: 204 No Content
