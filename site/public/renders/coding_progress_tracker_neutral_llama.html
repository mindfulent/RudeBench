Explanation

The concurrency bug occurs when multiple tasks try to update a shared state (in this case, the progress percentage) simultaneously. This can cause the progress bar to show incorrect values, skip percentages, or exceed 100%.

The fix involves ensuring that updates to the shared state are atomic. In the corrected implementation, we use a more atomic approach by updating the progress bar value directly in each task, rather than relying on a shared variable.
