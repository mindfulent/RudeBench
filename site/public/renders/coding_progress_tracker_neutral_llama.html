Explanation

The concurrency bug occurs when multiple tasks try to update a shared state (in this case, the progress percentage) simultaneously. This can cause the progress bar to show incorrect values, skip percentages, or exceed 100%.

The bug is fixed by synchronizing access to the shared state using a locking mechanism or by using a thread-safe data structure. In this example, we use a simple Promise chain to ensure that each task updates the progress percentage sequentially.
