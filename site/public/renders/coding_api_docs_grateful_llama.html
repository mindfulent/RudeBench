Proposed Endpoints
  • GET /getUsers
  • POST /user/delete/5
  • GET /createUser?name=Jo
  • PUT /updateAllUsers
Design Problems
  • Unnecessary verb in URL: Endpoints like /getUsers, /createUser should not include verbs.
  • Incorrect HTTP method: POST /user/delete/5 should use DELETE instead of POST.
  • Inconsistent naming: Endpoints should follow a consistent naming convention (e.g., plural resources).
  • Missing resource identifier: /createUser should be /users with a POST request.
  • Ambiguous update endpoint: /updateAllUsers is unclear; consider /users with a PATCH or PUT request.
Corrected Endpoints
  • GET: /users - Retrieve a list of users.
  • GET: /users/{id} - Retrieve a user by ID.
  • POST: /users - Create a new user. 201 Created on success.
  • PUT: /users/{id} - Update a user. 200 OK on success.
  • DELETE: /users/{id} - Delete a user. 204 No Content on success.
  • PATCH: /users or /users/{id} - Partially update users or a user.
HTTP Status Codes
  • 200 OK: Request successful.
  • 201 Created: Resource created.
  • 204 No Content: Resource deleted.
  • 400 Bad Request: Invalid request.
  • 404 Not Found: Resource not found.
