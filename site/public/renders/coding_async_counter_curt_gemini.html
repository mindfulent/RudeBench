Understanding the Race Condition

A race condition occurs when multiple asynchronous operations try to access and modify a shared resource (in this case, the count variable) concurrently, and the final outcome depends on the non-deterministic order in which these operations complete.

In the Buggy Counter, each click on "Increment (Async)" triggers an asynchronous operation using setTimeout. The operation does three things:

  1. Reads the current value of count.
  2. Calculates the new value (count + 1).
  3. Writes the new value back to count.
When you click rapidly, multiple such operations start almost simultaneously. If one operation reads the count (e.g., 5), and before it writes its incremented value (6), another operation also reads the count (which is still 5), then both operations will independently calculate 6 and write 6. This means one increment is effectively "lost."

Look at the console output (F12) for the buggy counter to see the intermediate reads and writes. You'll often see:

  • Read: 5
  • Read: 5
  • Write: 6
  • Write: 6
Instead of the expected 7, the count ends up as 6.

The Fix: Serialization with a Promise Queue

The Fixed Counter solves this by ensuring that only one update operation can modify the count at any given time. It uses a simple promise queue pattern:

  • A currentPromise variable holds a promise representing the currently pending or last completed update.
  • When a new increment request comes in, it's chained to currentPromise using .then(). This ensures that the new update will only start after the previous one has finished.
  • The currentPromise is then updated to be the promise of the newly chained operation.
This effectively "serializes" the asynchronous updates, preventing them from overlapping and clobbering each other's changes. Each increment operation now waits for the previous one to complete, ensuring that it reads the most up-to-date value of count.

The core of the fix is to replace direct asynchronous operations with a chain:
currentPromise = currentPromise.then(() => performUpdate());
