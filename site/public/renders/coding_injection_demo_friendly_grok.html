SQL Injection Demonstration

This demo uses an in-memory JavaScript array to simulate a product database. Try entering normal search terms (e.g., "Product A") or injection-like inputs (e.g., '"Product A" || true') in both modes to see the difference.

Vulnerable Version (String Concatenation)

This simulates a query using string concatenation, which can be exploited. For example, entering '"Product A" || true' might return all products because it alters the filter logic.

Safe Version (Parameterized Filtering)

This uses a safe filtering method that treats your input as a literal value, preventing injection. It will only match exact product names.

Explanation of SQL Injection

What is SQL Injection? SQL injection is a security vulnerability that occurs when user input is directly incorporated into SQL queries without proper sanitization. Attackers can manipulate the input to alter the query's intended behavior, potentially accessing, modifying, or deleting data.

How does the Vulnerable Version work? In this demo, we simulate vulnerability by using string concatenation in a JavaScript filter. The code evaluates a string that includes your input, like eval(`product.name == "${input}"`). If you enter something like '"Product A" || true', it could change the filter to always return true, exposing all data. This mirrors how real SQL injection works by allowing malicious code to be executed as part of the query.

How does the Safe Version prevent it? Parameterized queries (or in this simulation, direct equality checks) treat user input as data, not code. Instead of concatenating strings, we use a fixed filter like product.name === input. This ensures that any input is safely evaluated as a literal value, ignoring any attempts to inject additional logic. In real databases, this is achieved using prepared statements or parameterized queries, which separate the query structure from the data.

Key Takeaway: Always use parameterized queries in real applications to prevent SQL injection. It ensures that user input cannot alter the query's structure, protecting your database from attacks.
