Safe Version (Parameterized Queries)

    
    
    






    
Explanation

    
SQL injection occurs when an attacker is able to inject malicious SQL code into a web application's database in order to extract or modify sensitive data.

    
In the vulnerable version below, user input is directly concatenated into a SQL query string. This allows an attacker to inject malicious SQL.

    
In contrast, the safe version uses parameterized queries. The SQL query is defined with parameters, and then the user input is passed as a parameter to the query. This way, the database driver interprets the input as data, not as part of the SQL code, preventing injection.
