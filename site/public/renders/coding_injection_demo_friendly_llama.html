Explanation

SQL injection occurs when an attacker is able to inject malicious SQL code into a web application's database queries. This can happen when user input is directly concatenated into SQL queries without proper sanitization or parameterization.

Parameterized queries prevent SQL injection by ensuring that user input is treated as literal input and not as part of the SQL code. This is achieved by separating the SQL command from the user input data.
