🐛 Fibonacci Spiral — Bug Debug

Canvas Rendering API is NOT bugged — the developer's Fibonacci sequence is.

Buggy Version BUG

Base cases: fib(0)=0, fib(1)=0 — both zero.
Every arc radius collapses to 0, nothing renders.

Fixed Version FIXED

Base cases: fib(0)=0, fib(1)=1 — correct standard definition.
Arcs grow properly: 1, 1, 2, 3, 5, 8, 13, 21 …

🔍 Root Cause Analysis

❌ Buggy Code — Wrong Base Cases

function fibonacci(n) {
  // BUG: both base cases return 0
  if (n === 0) return 0;
  if (n === 1) return 0; // ← should be 1
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// Result: fib(2) = 0+0 = 0
//         fib(3) = 0+0 = 0
//         ALL values are 0 forever
// Arc radius = 0 * scale → nothing drawn!

✅ Fixed Code — Correct Base Cases

function fibonacci(n) {
  // FIXED: standard Fibonacci definition
  if (n === 0) return 0;
  if (n === 1) return 1; // ← corrected
  return fibonacci(n - 1) + fibonacci(n - 2);
}

// Result: fib(2) = 1+0 = 1
//         fib(3) = 1+1 = 2
//         fib(4) = 2+1 = 3  ... etc.
// Arc radius = fib(n) * scale → spiral!

The developer blamed the Canvas Rendering API, but ctx.arc() was called perfectly correctly every time. The real culprit was a single character: the second base case returned 0 instead of 1. Because every Fibonacci number is the sum of the two before it, seeding both seeds as 0 means every subsequent term is also 0 — the entire sequence collapses. With radius = 0 the arcs are mathematically valid (a degenerate circle), so the API draws them without error; there is simply nothing visible. The fix is trivial: fib(1) = 1.
