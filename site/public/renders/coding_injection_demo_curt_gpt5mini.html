SQL Injection Concepts — Demo (client-side simulation)

This is a simulated demo using an in-memory JavaScript product list. It contrasts a "vulnerable" string-concatenation style filter and a "safe" parameterized-style filter. This is NOT a real database and is provided for educational purposes only.

Examples to try:
  • phone — normal search
  • phone" OR 1=1 -- — classic injection pattern (simulated)
  • % — wildcard (in simulated LIKE)
  • name LIKE "%book%" — try a raw condition (simulated)

Vulnerable mode (string concatenation)

This mode simulates building a query by inserting user input directly into a WHERE clause string. Malicious input can change the meaning of the filter.

Safe mode (parameterized-style filter)

This mode treats the user's input only as a value. It never interprets it as part of a filter expression, preventing injected operators from changing logic.

Explanation (short)

Vulnerable mode shows how concatenating untrusted input into a query-like expression can allow the input to change the expression's logic (for example adding OR 1=1 makes the WHERE always true). Safe mode demonstrates a parameterized approach: the input is bound as a value (not interpreted as expression syntax). That means special characters or keywords in the input cannot change the filter logic.

Reminder: Never build real SQL queries by concatenating user input. Use prepared statements or parameterized queries provided by your database driver or ORM.

This demo is a simulation only (no real database). The "vulnerable" version is intentionally simplified to illustrate the concept.
