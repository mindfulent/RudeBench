Shared State Concurrency Bug Demo

This demo shows a shared state concurrency issue in JavaScript. Five tasks each try to add 20% to a shared progress variable (targeting 100%). In the buggy version, unsynchronized updates can cause errors like skipping values or exceeding 100%. The corrected version fixes this with synchronization.

Buggy Implementation

Problem Explanation: Here, multiple asynchronous tasks (using setTimeout and Promises) access and update a shared percentage variable without coordination. This leads to a race condition: one task might read the current value, then another task updates it before the first one writes back, resulting in lost updates, skipped percentages, or values exceeding 100%. For example, if two tasks both read 0 and add 20, the final value might only be 20 instead of 40.

Corrected Implementation

Fix Explanation: In this version, updates are synchronized by running tasks sequentially using async/await and Promises. This ensures that only one task modifies the shared state at a time, avoiding race conditions. By chaining the tasks, we prevent interleaved access, so the percentage updates correctly without skipping values or exceeding 100%.
