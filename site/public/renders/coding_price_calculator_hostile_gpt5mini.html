Explanation (no hedging)

JavaScript Number is a binary IEEE-754 floating point value. Decimals like 4.99, 0.10, 0.20 cannot be represented exactly in binary, so arithmetic produces tiny rounding errors (e.g. 4.99 becomes 4.989999999999999). Using === to compare totals or expecting decimal arithmetic to be exact is wrong and will silently break money computations. Use integers (cents) or an exact decimal library.
