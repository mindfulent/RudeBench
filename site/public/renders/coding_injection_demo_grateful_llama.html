Explanation

SQL injection occurs when an attacker is able to inject malicious SQL code into a web application's database in order to extract or modify sensitive data.

In the vulnerable example below, user input is directly concatenated into a SQL query. An attacker could input something like ' OR 1=1 -- to bypass authentication or extract unauthorized data.

To prevent SQL injection, use parameterized queries. These ensure that user input is treated as data, not executable code.
