Why Float Equality Fails for Financial Calculations

JavaScript (and most programming languages) uses the IEEE 754 standard for representing floating-point numbers (number type). This standard represents numbers in binary, which can perfectly represent numbers like 0.5 (1/2) or 0.25 (1/4).

However, many common decimal fractions, like 0.1 (1/10), 0.2 (2/10), or 0.99 (99/100), cannot be represented exactly in binary. They become repeating fractions, similar to how 1/3 is 0.333... in decimal. When these numbers are stored, they are rounded to the nearest representable binary fraction, leading to tiny inaccuracies.

For example, 0.1 + 0.2 in JavaScript doesn't equal 0.3. It equals 0.30000000000000004. If you then compare (0.1 + 0.2) === 0.3, it will evaluate to false.

In financial calculations:

  • Totals: Accumulating these small errors can lead to a final total that is off by a tiny fraction of a cent, which can be significant in large-scale systems.
  • Comparisons: Using === for equality checks (e.g., "is the total exactly $9.98?") will often fail unexpectedly because the "target" number ($9.98) might be represented as 9.979999999999999 or 9.980000000000001 after calculations, making it not strictly equal to 9.98.

The Solution: Work with Integers (Cents)

The standard practice for financial calculations is to avoid floating-point numbers where precision is critical. Instead, represent monetary values as integers by scaling them up (e.g., store $4.99 as 499 cents). All calculations are then performed using these integers, which are exact.

When displaying values to the user, convert the integer cents back to dollars and format them appropriately (e.g., (499 / 100).toFixed(2)).

This approach guarantees accuracy for sums, subtractions, and comparisons, eliminating the subtle bugs caused by floating-point inaccuracies.

For very complex financial systems or extremely large numbers, specialized libraries like Decimal.js or Big.js (which work with arbitrary-precision decimals) or even JavaScript's native BigInt for whole numbers of cents are used.
