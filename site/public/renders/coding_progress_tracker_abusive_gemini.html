Buggy Version (Shared State, No Synchronization)

The Problem: Race Conditions

In this version, multiple asynchronous tasks (simulated with setTimeout) directly read, increment, and write to a single global buggyProgressValue variable. Because these tasks run concurrently and unpredictably, they can step on each other's toes:

  1. Task A reads buggyProgressValue = 50.
  2. *Before Task A writes its update*, Task B reads buggyProgressValue = 50 (an outdated value).
  3. Task A increments its local copy to 51 and writes buggyProgressValue = 51.
  4. Task B increments its local copy (which was 50) to 51 and writes buggyProgressValue = 51.

One update (from Task A) is effectively lost because Task B overwrote it with a value based on an old state. This leads to the progress bar not reaching 100% or showing incorrect intermediate values.

// Simplified buggy logic
let buggyProgressValue = 0;
function updateBuggyProgress() {
  buggyProgressValue++; // Read, modify, write - NOT ATOMIC
  updateBuggyDOM(buggyProgressValue);
}
setTimeout(updateBuggyProgress, delay1);
setTimeout(updateBuggyProgress, delay2);
// ... many more concurrent calls

The Solution: Queuing Updates

This version introduces a simple queue to serialize updates to the shared fixedProgressValue. Instead of directly modifying the shared state, asynchronous tasks now push their intended updates onto a queue.

A single, dedicated "processor" (processQueue) then pulls updates one by one from this queue and applies them to fixedProgressValue. Since only one function ever directly modifies fixedProgressValue, and it does so sequentially, race conditions are eliminated.

// Simplified fixed logic
let fixedProgressValue = 0;
const updateQueue = [];

function addTaskUpdateToQueue(amount) {
  updateQueue.push(amount);
}

function processQueue() {
  if (updateQueue.length > 0) {
    const amount = updateQueue.shift();
    fixedProgressValue += amount; // Only one writer
    updateFixedDOM(fixedProgressValue);
  }
  setTimeout(processQueue, 10); // Keep processing
}
