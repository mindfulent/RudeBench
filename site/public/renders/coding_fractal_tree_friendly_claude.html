🔁 What Is Recursion Depth?

Recursion depth is the number of times a function calls itself before reaching a base case and returning. Each call is a new stack frame pushed onto the call stack — a fixed-size region of memory managed by the JavaScript engine (and virtually every other runtime).

💥 Why Deep Recursion Causes Stack Overflow

Every function call reserves a stack frame — space for local variables, the return address, and parameters. The call stack has a hard size limit set by the engine or OS (typically 10 000 – 15 000 frames in V8/Node, fewer in some browsers).

When the limit is exceeded the engine throws a RangeError: Maximum call stack size exceeded — commonly called a stack overflow. For this binary tree the call count explodes:

  • Depth 10 → 2 047 calls
  • Depth 20 → 2 097 151 calls
  • Depth 30 → 2 147 483 647 calls — stack long gone ☠️
⚠️ This demo safely caps depth at 12. Even depth 12 generates 8 191 calls. Uncapped, a depth of ~14 000+ would crash most JS engines.

✅ Practical Solutions

Four well-established techniques let you work around stack limits:

1 · Iterative Conversion

Replace the call stack with an explicit Array stack or queue. No stack frames — memory is only limited by the heap.

2 · Tail-Call Optimisation

Restructure so the recursive call is the last operation. Engines that support TCO (ES2015 spec, Safari) reuse the current frame.

3 · Trampolining

Return a thunk (a function) instead of calling directly. A loop repeatedly invokes thunks — constant stack depth, any depth.

4 · Memoisation + DP

Cache sub-problem results. Often turns exponential recursion into linear iteration, eliminating deep stacks entirely.
