Progress Bar Concurrency Bug Demo

Simulated "concurrent" tasks update a shared progress value. Left is intentionally buggy (updates race). Right is fixed by serializing updates (queueing) and clamping the value.

Buggy version

Progress: 0%
Expected total from tasks: 100%. Buggy updates may exceed 100% or show odd steps.
Why it breaks: Each task reads the shared percentage, computes a new value, then writes it. Because the reads/writes happen in asynchronous callbacks, two callbacks can read the same old value and both write increments, causing overcounting.

Fixed version

Progress: 0%
Fix: serialize updates using a simple Promise-based queue (mutex) and clamp to 100%.
How it fixes: Every update is applied only after previous updates complete, preventing interleaving where two tasks both base their change on the same stale value.

Parameters

Adjust values to make overcounting more likely (e.g. many tasks and each pct summing >100).

Short code excerpts

/* Buggy update (simplified) let pct = 0; asyncTask(() => { const read = pct; // read const next = Math.min(100, read + inc); // compute pct = next; // write }); */
/* Fixed update: queue updates so they don't interleave let pct = 0; let queue = Promise.resolve(); function enqueueUpdate(fn){ queue = queue.then(() => fn()); return queue; } asyncTask(() => enqueueUpdate(() => { pct = clamp(pct + inc,0,100); })); */
