Why Parameterized Queries Work

The core problem with SQL Injection (and similar injection attacks in other contexts) is that the application treats user-supplied data as part of the executable code (the query).

1. The Vulnerable Approach (String Concatenation):

When you build a query by directly concatenating user input into a SQL string, like: "SELECT * FROM products WHERE name = '" + userInput + "'" the database engine sees one single string. If userInput is ' OR 1=1 --, the resulting query becomes: SELECT * FROM products WHERE name = '' OR 1=1 --'. The -- turns the rest of the original query into a comment, and OR 1=1 is always true, effectively bypassing the original filter and returning all records.

The database has no way to distinguish between the intended data and the injected code. It just executes the final, combined string.

2. The Safe Approach (Parameterized Queries):

With parameterized queries (or prepared statements), you define the structure of the query first, marking placeholders for where data will go: SELECT * FROM products WHERE name = ? (or :name, @name depending on the database system).

Then, you provide the user input separately as a parameter for that placeholder. The database engine processes these two parts distinctly:

  • It first parses and compiles the query structure (SELECT * FROM products WHERE name = ?). It understands that ? is a placeholder for a value.
  • Then, it takes the user input (e.g., ' OR 1=1 --) and treats it *only* as a literal string value to be inserted into that placeholder. It automatically escapes any special characters within the user input.

So, if userInput is ' OR 1=1 --, the database will treat the *entire string* ' OR 1=1 -- as the value for the name column. It will look for a product whose name is literally ' OR 1=1 --, which is highly unlikely to exist, and thus no injection occurs.

The key takeaway: Parameterized queries ensure that user input is always treated as data, never as executable code. This separation of concerns is fundamental to preventing injection attacks.
