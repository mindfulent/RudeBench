Why "the garbage collector isn't working" is almost always the wrong diagnosis

⚠️ The Real Problem: Retained References, Not a Broken GC

JavaScript's garbage collector works by freeing objects that are no longer reachable from any live root (global scope, call stack, closures, etc.). When memory grows after removing widgets, the GC is doing its job perfectly — it simply cannot collect objects that still have references pointing at them.

  • Buggy version: Each removed widget still lives in a global leakyRegistry array (DOM reference + closure + interval handle). The GC sees live references → it must keep the objects alive. Memory grows forever.
  • Fixed version: Before removal, every listener is detached with removeEventListener, every interval is cleared with clearInterval, and the registry entry is deleted. No more live references → GC can collect freely.
  • The "simulated memory" counter below models the retained object graph, not performance.memory (which requires special browser flags).
