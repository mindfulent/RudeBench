Understanding Vulnerable Mode

                
In Vulnerable Mode, the user's input is directly inserted into the query string using simple string concatenation. This allows malicious input to change the structure and intent of the query.

                
Risk: An attacker can inject SQL keywords, logical operators, or comments to bypass authentication, retrieve unauthorized data, or even modify/delete data.

                
Example Injection: Try searching for ' OR 1=1 -- (without the outer quotes). This will cause the OR 1=1 condition to always be true, effectively returning all products, and the -- comments out the rest of the original query.

            

        


        

            
Why Parameterized Queries Are Safe

            

                
The Solution: Parameterized Queries

                
Parameterized queries (or prepared statements) are the standard and secure way to handle user input in database operations. They work by:

                
    
                    
  1. Separating Query Structure from Data: The database query is defined with placeholders (parameters) for user input.
    2. 
                    
  2. Binding Parameters: The user's input is then "bound" to these placeholders. The database engine treats these bound values strictly as data, not as executable code.
    3. 
                    
  3. Escaping Handled Automatically: The database driver automatically handles any necessary escaping of special characters within the bound data, preventing them from being interpreted as part of the query's syntax.
    4. 
                

                
Benefit: Even if a user inputs malicious characters like ', OR, or --, they are treated as literal parts of the search term, not as commands to modify the query logic. The query's structure remains fixed and controlled by the developer.

                
Think of it like this: Instead of building a sentence by literally inserting words into another sentence, you're filling out a form. The form has specific fields, and whatever you type into a field is treated only as the content for that field, not as instructions to change the form itself.
