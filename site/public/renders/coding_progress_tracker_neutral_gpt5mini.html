Progress Bar Concurrency Bug Demo

Two side-by-side demos: left shows a buggy approach where tasks race when updating a shared percentage variable; right shows a corrected approach that avoids the race. Use "Start" to run a single simulation or "Run 10 times" to see the bug appear more often.

Buggy Implementation
0%
Problem: multiple asynchronous tasks read and write the same shared percentage variable without coordination. Race conditions can make the progress jump, skip values, or even exceed 100%.
Fixed Implementation
0%
Fix: each task reports its own share of work; the aggregator performs a single atomic update (adding the task's contribution). Alternatively, serialize updates or use per-task bookkeeping to avoid overwriting reads from other tasks.
