Improved REST API Documentation

This page documents a corrected version of your API, addressing issues with the original design (e.g., GET /getUsers, POST /user/delete/5, GET /createUser?name=Jo, PUT /updateAllUsers). We've applied RESTful conventions to make it more standard, using resource-based URLs, appropriate HTTP methods, and proper request/response structures.

Key Improvements:

  • Replaced verbs in URLs with nouns (e.g., /users instead of /getUsers).
  • Used correct HTTP methods for actions (e.g., DELETE for deletions, POST for creations).
  • Utilized path parameters for resource IDs and request bodies for data.
  • Avoided unsafe practices like using GET for modifications.

Available Endpoints

GET /users

Description

Retrieves a list of all users. This replaces the original GET /getUsers endpoint.

Request

  • Method: GET
  • Path: /users
  • Query Parameters: None required (optional for filtering, e.g., ?limit=10)
  • Request Body: None

Response

  • Status Code: 200 OK
  • Body: JSON array of users, e.g., 
    [{ "id": 1, "name": "Alice" }, { "id": 2, "name": "Bob" }]

Fix: Removed the verb 'get' from the path for better REST compliance.

POST /users

Description

Creates a new user. This replaces the original GET /createUser?name=Jo endpoint, which incorrectly used GET and query parameters.

Request

  • Method: POST
  • Path: /users
  • Headers: Content-Type: application/json
  • Request Body: JSON object, e.g., 
    { "name": "Jo" }

Response

  • Status Code: 201 Created
  • Body: JSON of the created user, e.g., 
    { "id": 5, "name": "Jo" }
  • Status Code: 400 Bad Request (if input is invalid)

Fix: Switched to POST for creation and moved data to the request body instead of query parameters.

DELETE /users/{id}

Description

Deletes a specific user by ID. This replaces the original POST /user/delete/5 endpoint, which used the wrong method and path structure.

Request

  • Method: DELETE
  • Path: /users/{id} (e.g., /users/5)
  • Request Body: None

Response

  • Status Code: 204 No Content (successful deletion)
  • Status Code: 404 Not Found (if user doesn't exist)

Fix: Used DELETE method and a proper resource path with the ID as a path parameter.

PUT /users/{id}

Description

Updates a specific user by ID (replaces the entire resource). This addresses the original PUT /updateAllUsers endpoint, which was unclear and non-standard. For bulk updates, consider using PATCH /users instead.

Request

  • Method: PUT
  • Path: /users/{id} (e.g., /users/5)
  • Headers: Content-Type: application/json
  • Request Body: JSON object with updated data, e.g., 
    { "name": "Jo Updated" }

Response

  • Status Code: 200 OK
  • Body: JSON of the updated user, e.g., 
    { "id": 5, "name": "Jo Updated" }
  • Status Code: 404 Not Found (if user doesn't exist)
  • Status Code: 400 Bad Request (if input is invalid)

Fix: Focused on updating a single resource. For the original bulk update intent, a better approach might be PATCH /users for partial updates of multiple resources, but that's not implemented here without more details.
