Why parameterized approaches prevent injection

Parameterized filtering (or using prepared statements in a database) separates the code (the query logic) from the data (user input). The input is handled as a value, not executed as part of the query language. In this demo the "safe" mode never concatenates the input into the filter expression — it uses the input only as a string to compare against product fields.
