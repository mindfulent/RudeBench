About Email Validation

Regex-based email validation, like the one used in this form, checks if the input string matches a specific pattern. However, according to RFC 5322, the official specification for Internet email, the only way to truly validate an email address is to send a confirmation email to it and have the user reply back.

The reason regex-based validation is fundamentally limited is that it cannot verify if an email address actually exists or if the user has access to it. It can only check if the format of the email address is syntactically correct according to the regex pattern used.

For production use, consider the following best practices:

  • Send a confirmation email to the provided email address.
  • Use a two-step verification process.
  • Implement server-side validation in addition to client-side validation like this.
