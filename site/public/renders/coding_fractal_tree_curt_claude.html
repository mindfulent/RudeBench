📚 Understanding Recursion & Its Limits

🔍 How the Fractal Tree Uses Recursion

Each call to drawBranch(x, y, angle, length, depth) draws one line segment, then calls itself twice — once for the left sub-branch and once for the right — with a shorter length and adjusted angle. The recursion stops when depth === 0. 

function drawBranch(x, y, angle, length, depth) {
  if (depth === 0) return;          // base case

  const x2 = x + Math.cos(angle) * length;
  const y2 = y + Math.sin(angle) * length;

  ctx.beginPath();
  ctx.moveTo(x, y);
  ctx.lineTo(x2, y2);
  ctx.stroke();

  drawBranch(x2, y2, angle - SPREAD, length * RATIO, depth - 1); // left
  drawBranch(x2, y2, angle + SPREAD, length * RATIO, depth - 1); // right
}

At depth d, the total number of branches is 2d+1 − 2. Depth 12 produces 8 190 branches — perfectly manageable because the call stack only needs to hold one path from root to leaf at a time (depth 12 = 12 frames deep).

Notice that even at depth 12 the stack is only 12 frames deep. The tree is wide (many branches) but not tall in stack terms — this is why binary-tree recursion rarely overflows in practice unless the depth is enormous.

💥 Why Deep Recursion Causes Stack Overflow

Every function call pushes a stack frame onto the call stack. The frame stores the return address, local variables, and parameters. JavaScript engines typically allow between 10 000 and 15 000 frames before throwing: 

// Naive countdown — blows the stack for large n
function countdown(n) {
  if (n === 0) return "done";
  return countdown(n - 1);   // each call waits for the next
}

countdown(100000);
// ❌ RangeError: Maximum call stack size exceeded
  • Each pending call occupies memory until it returns.
  • Linear recursion (f(n) → f(n-1)) keeps n frames alive simultaneously.
  • The fractal tree only keeps depth frames alive — much safer.
  • Stack size is fixed by the engine (V8 ≈ 15 000 frames on 64-bit systems).

🛠️ Practical Solutions

① Iterative Approach (Explicit Stack)

Replace the call stack with a manual stack stored in a plain array. Zero risk of overflow.

function drawTreeIterative(startX, startY, startAngle, startLen, maxDepth) {
  const stack = [{ x: startX, y: startY,
                    angle: startAngle, len: startLen, depth: maxDepth }];

  while (stack.length > 0) {
    const { x, y, angle, len, depth } = stack.pop();
    if (depth === 0) continue;

    const x2 = x + Math.cos(angle) * len;
    const y2 = y + Math.sin(angle) * len;
    drawLine(x, y, x2, y2);

    stack.push({ x: x2, y: y2, angle: angle - SPREAD, len: len * RATIO, depth: depth - 1 });
    stack.push({ x: x2, y: y2, angle: angle + SPREAD, len: len * RATIO, depth: depth - 1 });
  }
}

② Tail-Call Optimisation (TCO)

A function is tail-recursive when its recursive call is the very last operation — the engine can reuse the current frame instead of pushing a new one, keeping stack depth at O(1). 

// Tail-recursive accumulator pattern
function sumTo(n, acc = 0) {
  if (n === 0) return acc;
  return sumTo(n - 1, acc + n);  // ← tail position ✅
}
// ES2015 specifies TCO, but only Safari fully implements it (2024).
// In V8/SpiderMonkey, use the iterative form for safety.

③ Trampolining

A trampoline is a loop that repeatedly calls a function that returns either a value or another function to call. It converts deep recursion into a flat loop without changing the logical structure much. 

const trampoline = fn => (...args) => {
  let result = fn(...args);
  while (typeof result === "function") result = result();
  return result;
};

const countdown = trampoline(
  function step(n) {
    if (n === 0) return "done";
    return () => step(n - 1);  // return thunk, not a call
  }
);

countdown(1_000_000); // ✅ no overflow

④ Memoisation & Dynamic Programming

For problems with overlapping sub-problems (e.g. Fibonacci, shortest paths), cache results so each sub-problem is solved only once — reducing both time complexity and the depth of the recursion tree.
